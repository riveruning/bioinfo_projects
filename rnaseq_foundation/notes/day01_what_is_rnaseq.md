# Day 1: What Is an RNA-seq Project?

## 1. 今日问题

RNA-seq 项目到底在分析什么？

## 2. 我的理解

RNA-seq 分析的目标不是直接“看 FASTQ 文件”，而是把测序产生的大量 reads 转化成基因表达信息。

一个完整的 RNA-seq 项目通常包括：

1. 明确生物学问题；
2. 设计实验分组；
3. 获得 RNA 测序数据；
4. 检查 FASTQ 原始数据质量；
5. 将 reads 比对到参考基因组，或者直接定量到转录本；
6. 得到 count matrix；
7. 使用统计方法寻找差异表达基因；
8. 对差异基因进行生物学解释。

## 3. FASTQ 不是最终结果

FASTQ 文件只记录 reads 的序列和质量分数。

它告诉我们：

- 测到了哪些序列；
- 每个碱基的测序质量如何。

但它不直接告诉我们：

- 这些 reads 来自哪个基因；
- 每个基因表达量是多少；
- 哪些基因发生了差异表达；
- 哪些生物学通路发生了变化。

## 4. 表达矩阵是统计分析的起点

count matrix 记录每个基因在每个样本中的 read counts。

例如：

| gene_id | control_1 | control_2 | treat_1 | treat_2 |
|---|---:|---:|---:|---:|
| GeneA | 120 | 135 | 560 | 600 |
| GeneB | 45 | 39 | 41 | 50 |
| GeneC | 0 | 1 | 0 | 0 |

后续的 PCA、差异表达分析、火山图、热图和富集分析都依赖表达矩阵。

## 5. 差异表达分析想回答什么？

差异表达分析想回答：

某个基因在两组样本之间的表达差异，是否大到不像随机波动？

例如：

- control 组中 GeneA 表达较低；
- treatment 组中 GeneA 表达明显升高；
- 如果这种升高在多个生物学重复中都稳定出现，那么 GeneA 可能是 treatment 响应相关基因。

## 6. 今天的核心结论

RNA-seq 项目的核心路线是：

FASTQ → 质量控制 → 比对或定量 → count matrix → 差异表达 → 生物学解释。

我现在要学的不是单个软件，而是如何把这条路线完整走通。
